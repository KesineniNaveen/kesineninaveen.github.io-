Introduction

Power BI Desktop is a tool to connect, transform, model, and visualize data. Use Power BI Service to publish and share dashboards.

Power Query (M)

Used for ETL: extract, transform, load. Common tasks: merging queries, pivot/unpivot, data type conversion.

// Example: remove top rows in M

  Source = Excel.Workbook(File.Contents("file.xlsx"), null, true),

  RemovedTop = Table.Skip(Sheet,4)

  RemovedTop

Data Modeling

Star schema recommended: fact table + dimension tables. Create relationships on keys, set cross-filter direction carefully.

  • Fact table: transactional data
  • Dimension tables: products, customers, dates
  • Use surrogate keys where necessary

DAX Basics

Calculated columns vs measures. Use CALCULATE to change filter context.

Total Sales = SUM(Sales[Amount])
Sales LY = CALCULATE([Total Sales], SAMEPERIODLASTYEAR('Date'[Date]))

Advanced DAX

  • Time intelligence (TOTALYTD, SAMEPERIODLASTYEAR)
  • Maintain filter context with ALL, ALLEXCEPT
  • Use variables for complex calculations

Visuals & Formatting

Choose visuals that match the story. Use bookmarks & drill-through for interactivity. Keep visuals minimal and use consistent colors.

Performance Tips

  • Reduce columns in the data model
  • Avoid calculated columns where possible
  • Use measures and aggregations
  • Manage relationships and cardinality

Deployment

Publish to Power BI Service, configure refresh, set workspace permissions, and use Power BI apps for distribution.
